# Notes

雑多なメモ。
主に現場で感じたことや暗黙知っぽいやつを記載していく。<br>
時と場合によって最適解が変わるので、あくまで参考程度に。
とりあえず殴り書き。あとで整理する。

| 用語 | 詳細 |
| --- | --- |
| 進捗の共有 | まずパーセンテージにすると何％くらい終わっているのかを共有すると、相手はイメージしやすい。 |
| 図を使う | 誰かに何かを説明するときは図を使う。<br>エクセルの図形でサクッと作ったものでもいいので図があるとかなり理解の助けになる。 |
| 行数のわかるエディタを利用する | メモ書きや、テキストを画面共有する場合に、相手がどの行を見ているのかを把握しやすくなる。 |
| 作業を俯瞰する | 今やっていることは目的のためになっているか、優先順位が違わないか、など都度確認。<br>作業に没頭しすぎて視野が狭くならないようにする。<br>ある程度進んでから「これ今やらなくてもいいじゃん」となったりしないためにも。自分の場合は検討や調査で起きがち。 |
| 見積もり | その現場の作業にある程度なれないとわからないところもあるが、見積もりのブレを少なくする努力をする。見積もり後に作業が増えた場合はその追加作業で見積もりがどのように変わるかを共有する。「頑張れば元の見積もりポイントでもこなせるから言わない」というのは自分の首を絞めることになる。 |
| スキル≠コーディング | コーディングがスキルの大半、と思いきや、実際にコーディングをするための準備の要素が結構な割合を占める。<br>設計や要件定義、使用の確認、シーケンス起こし等々、しっかり固めてようやくコーディングができる状態になる。<br>つまりコーディングだけできても市場価値があまり上がらない＆生成AIの登場で立場が危うい。 |
| レビューの重要性 | 指摘を受けるのは精神的に消耗しがちなので後回しにしがちだが、見てもらえる機会があるたびに見せたほうが結果的に遠回りしなくて済んだり、間違いに早い段階で気づくことができる。途中でもある程度見せれる状態なら見せよう。 |
| マイク | オンライン会議の時にマイクが周りの音を拾ってしまうと、とっても聞き取りづらくなり相手にかなり不快感を与える。<br>静かな場所で話すか、ノイズキャンセリング機能のあるマイクを使うと良い。 |
| イヤホン | 高性能なワイヤレスタイプも普及しているが、なんやかんや有線イヤホンのほうが安心感がある。<br> ワイヤレスマイクはBluetoothが切れたりした際に焦るし、ペアリングしなおしたりする手間がある。 |
| 気難しい人 | ひとこと余計だったり、言い方がチクチクしたりする人もいる。<br>俺からしたら信じられないが、そんなんでも一切悪気がない人もいるようなので気にしすぎないでおく。また、立場的に板挟みだったり、そうせざるを得ない事情などもあったりするので想像力をもって寛大に受け止める。<br>チクチク言われないためにも自己研鑽。 |
| 会議でしゃべる | 意外と緊張する。知らない用語が出てこないか、議論で渡り合えるか、などなど。<br>どうにもならないこともあるが、何について話す会議か、何がゴールかを明確にすると、ある程度は用語を調べておいたり、カンペ作ったり準備できる。 |
| 入場したら... | 自分は何の業界の何のシステムにかかわるのかを把握する。<br>様々なシステムと連携していて、その一部を担当する場合は、どの位置に属して何をするのかをしっかり把握する。<br>業界用語、業務知識が必要になる場面も出てくるので、少しずつ普段から調べておいたりメモを蓄積しておく。 |
| あいさつ| 社内の雰囲気によって誰にでもいうかどうか、みたいなところはあるが、基本的には自分からあいさつする。別の島や他部署の人で挨拶しにくさがあったら頭でも下げとく。 |
| 雑談 | 個人的に機会があったらしとくのが得だと思っている。<br>仕事以外で距離を縮めておくと、仕事関連の相談する心理的障壁がお互いに下がる。あと気にかけてもらえたりする（言葉にすると何だかいやらしいが...） |
